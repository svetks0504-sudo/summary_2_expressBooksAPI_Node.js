Создайте Express сервер для управления коллекцией книг. Сервер должен поддерживать различные типы запросов (GET, POST, PUT, PATCH, DELETE) и работу с параметрами маршрутов и query-параметрами.

Требования:
Создайте проект expressBooksAPI и установите Express
Храните данные о книгах в массиве в памяти (не используйте базу данных)

Структура книги:
{
  id: число (автоинкремент),
  title: строка,
  author: строка,
  year: число (год издания),
  genre: строка,
  isAvailable: boolean (доступна ли книга)
}
Создайте следующие маршруты:
GET /books – возвращает список всех книг. Поддерживает query-параметры:
?genre=фантастика – фильтрация по жанру
?author=Толстой – фильтрация по автору
?available=true – фильтрация по доступности
?limit=5&offset=10 – пагинация
GET /books/:id – возвращает книгу по ID. Если книга не найдена – статус 404
GET /books/stats/genres – возвращает статистику: количество книг по каждому жанру
POST /books – добавляет новую книгу. Принимает JSON с полями title, author, year, genre, isAvailable. Валидирует данные (все поля обязательны, year должен быть от 1500 до 2025)
PUT /books/:id – полностью заменяет книгу по ID. Принимает все поля. Если книга не найдена – статус 404
PATCH /books/:id – частично обновляет книгу (только переданные поля). Если книга не найдена – статус 404
DELETE /books/:id – удаляет книгу по ID. Если книга не найдена – статус 404
POST /books/:id/borrow – отмечает книгу как выданную (isAvailable = false)
POST /books/:id/return – отмечает книгу как доступную (isAvailable = true)

Добавьте middleware для:
Логирования всех запросов (метод, URL, тело запроса для POST/PUT/PATCH)
Обработки ошибок валидации
Обработки 404 для несуществующих маршрутов
